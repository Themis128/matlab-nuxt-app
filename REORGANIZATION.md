The project root has been reorganized for better maintainability:

```

├── setup_matlab_env.m
├── analyze_mobiles_dataset.m
├── extract_all_insights.m
├── generate_visualizations.m
└── run_all_examples.m

scripts/             # Shell, batch, and utility scripts
├── utils/           # JavaScript/Python utilities
│   ├── check-capabilities.js
│   ├── check-capabilities.py
│   ├── run-next-steps.js
│   ├── view_mat_file.js

├── ENVIRONMENT_SETUP.md

├── EXAMPLES_SUMMARY.md

└── matlab-capabilities-report.txt

data/                # Datasets
└── Mobiles Dataset (2025).csv (moved from root)
```

## Updated References

### Package.json Scripts

- `npm run check` → now runs `scripts/utils/check-capabilities.js`
- `npm run mat:view` → now runs `scripts/utils/view_mat_file.js`
- `npm run next-steps` → now runs `scripts/utils/run-next-steps.js`

### MATLAB Scripts

When calling MATLAB scripts, use:

```matlab
run('matlab/setup_matlab_env.m')
run('matlab/analyze_mobiles_dataset.m')
```

### Python Scripts

Utility scripts maintain their relative paths from project root.

## Removed Files

- Temporary files: `temp_predict_*.m`, `test_column_names.m`
- Obsolete configs: `.eslintignore`, `.eslintrc.cjs` (using flat config)
- `test-results.json` (regenerated by tests)
